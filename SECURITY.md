# Security Policy

## Supported versions

Only the latest `main` branch is currently supported for security fixes.

## Reporting a vulnerability

Please report vulnerabilities privately using GitHub Security Advisories:

1. Open the repository on GitHub.
2. Go to `Security` > `Advisories`.
3. Create a private advisory with reproduction details and impact.

Do not open public issues for security vulnerabilities.

## What to include

1. A clear description of the issue and potential impact.
2. Reproduction steps or a proof of concept.
3. Affected files, endpoints, or tool methods.
4. Any temporary mitigations you identified.
